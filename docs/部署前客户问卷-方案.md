# 部署前客户问卷 · 方案（问卷星可用）

> 用途：一次填表，拿到"部署前 + 后续运维会影响设计/预期/成本"的关键信息，后续少返工。
> 填写对象：客户方老板（非技术）。语言已做高情商打磨：少术语、低认知成本、每题附"为什么问"以建信任，并为服务器/托管成本做心理预期铺垫。
> 跨部门打磨过程见同目录临时纪要（看完可删）。

## 三个版本怎么选
| 版本 | 题量 | 适用 |
|---|---|---|
| **精简版** | 12 题（仅【必】） | 老板很忙、只想快速对齐核心；5 分钟填完。后续不足的再单聊补。 |
| **标准版（推荐）** | 22 题（【必】+【标】） | 一次把设计、验收、运维、成本都对齐，颗粒度刚好；约 8-10 分钟。 |
| **完整版** | 27 题（全部） | 想"一次到位、后续所有问题都有依据"；约 12 分钟。 |
> 题目下方标签：【必】=三版都含；【标】=标准版起含；【全】=仅完整版含。按选定版本删去更高标签的题即可。

## 问卷星录入提示
- 单选→**单选题**；多选→**多选题**；单行填空→**填空题**；多行填空→**问答题(多行)**；矩阵→**矩阵单选题**（行/列已给）。
- 矩阵题需在问卷星编辑器手动设行与列；其余可用"文本快速导入"粘贴后微调。
- 建议开启**分页**（每分节一页）+ 进度条，降低老板填写压力。

## 一、先认识您的生意（破冰与基本盘）

**1.【必｜多选+填空】** 上线后大概有几位同事会用这套系统？请勾选角色并在后面填人数：
- 业务员对外报价（__人） / 专人维护产品和价格（__人） / 您本人或主管把关·看汇总（__人） / 内勤或其他（__人）
> 💬 业务员、价格维护、老板审批是三种不同的使用面，提前问清是想把界面和权限配顺，也不让人人看到不该看的报价金额。

**2.【必｜矩阵单选】** 帮我们大致估个量级就行，不用精确：
- 行：产品/型号数量 ｜ 部件或采购件数量 ｜ 供应商数量 ｜ 每月新增报价单数量 ｜ 平时同时在线使用人数
- 列：几十 ｜ 几百 ｜ 上千 ｜ 几千以上 ｜ 不清楚
> 💬 这几个量级直接决定我们给您配多大的服务器，也就决定每月那笔托管费。先有个大概，我们才能把成本稳稳压在百元级——不买大了浪费、也不买小了卡顿。

**3.【必｜问答】** 现在一张报价单是怎么出来的？（用什么工具：Excel/Word/口头/旧系统；价格从哪查；要不要谁确认才发客户）其中最头疼、最易出错或最耗时的是哪一步？
> 💬 新系统的价值不是把现流程原样搬上电脑，而是把最痛那一步解决掉。了解现状，才能保证您切换后明显变轻松。

**4.【必｜矩阵单选】** 凭感觉，下面几件事各自多久发生一次：
- 行：查找现有产品和价格 ｜ 配置新规格产品 ｜ 给客户出报价单 ｜ 改动/新增价格 ｜ 报价发出前需您过目
- 列：每天多次 ｜ 每天 ｜ 每周 ｜ 每月 ｜ 几乎不
> 💬 想知道哪件是一天点几十次的"主干道"，好把精力压在最常用那一两步、让它快到极致。

**5.【标｜单选】** 您家产品，一个型号通常有很多可选规格组合吗？
- 基本一个型号一款 / 一个型号有几种规格 / 一个型号几十种组合 / 组合很多、上百上千 / 不太确定
> 💬 这套系统的核心就是把"可配置的规格组合"管起来。几十种还是上千种，决定配置与查重要做多重。

**6.【标｜填空】** 未来一年，有没有扩人或开新业务线的打算？用报价这块的业务员预计会增加吗？
> 💬 想合理预留余量、按您的节奏配资源，既不卡顿也不让您为暂时用不到的容量多花钱。

## 二、上线节奏与里程碑

**7.【必｜单选】** 您心里有希望正式用起来的时间点吗？是否有绕不开的硬节点（旺季/展会/招标）？
- 有硬节点，过了有实际影响（备注写大概时间） / 有希望时间但可商量 / 尽快就好、不卡死日子 / 还没想好
> 💬 时间是"希望"还是"必须"，决定我们怎么倒排进度。坦白说时间越紧、要搬的旧数据越多，我们越要提前备人，这也关系到交付节奏的报价。

**8.【必｜单选】** 贵公司打算用哪个网址访问系统？域名是已有还是需新注册？之前给网站做过"ICP 备案"吗（工信部要求的企业网站对外开放前的登记手续）？
- 已有域名且已备案 / 已有域名但没备案 / 需新注册域名 / 不清楚，需要协助
> 💬 境内云上线，按规定企业网站对外开放前必须备案，这是整个流程里唯一"快不了、只能等审批"的环节（通常 3–22 个工作日）。其余部署、测试、搬数据我们都能在等备案期间并行做完——所以早确认，您的上线时间才稳。

**9.【必｜单选】** 上线后平时主要您本人把关，还是指定一位同事日常对接？这位对接人是否也是最终能拍板"可以正式用了"的人？
- 我本人既对接也拍板 / 指定同事对接，我本人最终拍板 / 指定同事既对接也能拍板 / 还有合伙人/财务等需一起参与
> 💬 提前认对人，沟通确认能第一时间找到您、不耽误时间，也避免方案做到一半被没露面的同事推翻。

**10.【全｜单选】** 新系统上线时，您希望"选个不忙的时间、旧的停新的开、一次切过去"，还是"新旧并行一小段、确认没问题再切"？
- 选不忙时间一次切 / 新旧并行一段再切 / 听你们建议 / 还没想好
> 💬 上线那一下对业务多少有影响。知道您倾向，我们好选影响最小的窗口、给彼此留兜底余地。

## 三、把"命根子"数据管好

**11.【必｜单选】** 关于报价金额，您更倾向哪种管法？（业务员之间该不该看到彼此报给客户的价，可在备注说明）
- 业务员能自己定价、直接发客户 / 业务员可报价但发出前要您或主管过目 / 价格由专人维护、业务员只能选用不能改 / 还没想好，想听建议
> 💬 您提过报价金额是敏感数据。这题直接决定系统是"敞开"还是"有把关、有隔离"，先问清免得上线后权限太松或太紧又返工。

**12.【标｜多选】** 系统里会存报价金额和客户联系人这些压箱底的信息。您最担心的是什么？
- 怕被同行/竞对看到 / 怕业务员离职把客户和报价带走 / 单纯怕数据丢失、想稳妥 / 担心合规、个人信息出问题 / 暂时没特别担心的
> 💬 这是您最在意的事，我们主动问是想替您把关。怕泄密、怕带走、怕丢失对应不同防护做法——您说出来我们好精准做到您放心，并按个人信息保护要求来管。

**13.【全｜多选】** 这套报价系统，您希望它独立用就好，还是要和别的系统连起来？
- 独立使用即可 / 报价金额希望导到财务·记账系统 / 要和现有 ERP·进销存对接 / 希望用企业微信·钉钉账号直接登录 / 暂不确定，先独立用以后再说
> 💬 独立使用和对接其他系统是工作量完全不同的两种方案。提前知道边界，避免上线后才发现要对接、临时返工拖期。选"以后再说"也没问题，我们会预留接口。

## 四、旧数据怎么搬过来

**14.【必｜多选】** 您现在的产品、报价数据主要存在哪里？有没有懂这套系统的同事能配合我们把数据导出来？
- 一套专门软件/系统（备注写名字） / Excel 表格台账 / 纸质单据，基本没电子化 / 业务员各自手里的表，没统一 / 其他
> 💬 这是数据能否顺利搬家的总开关。您只需说清数据在哪、有没有人能配合导出，字段级技术细节我们后续直接对接贵司懂系统的同事，不麻烦您。

**15.【必｜问答】** 旧数据迁移由哪位同事对接、大概什么时间段能配合？旧数据能以什么形式给我们（导出的 Excel / 数据库导出文件 / 只能登录旧系统人工查看）？
> 💬 数据迁移最容易卡在"拿不到数据"或"对接人没空"。早点知道谁配合、何时有空、数据怎么给，我们好把这段时间提前锁进计划，避免临到上线才发现搬不动。

**16.【全｜问答】** 您家产品/型号有没有一套自己的编号规则（固定前缀、按类别编号）？上线后想沿用还是借机重新规范？报价有没有一定要保留的特殊习惯（必须区分含税/不含税、客户专属价）？不确定可简单写两句。
> 💬 编号规则和含税口径是这行容易踩坑的地方。提前知道您的习惯，迁移时就能对齐，避免老编号对不上、含税口径搞混导致金额出错。

## 五、验收与上线后怎么配合

**17.【必｜问答】** 上线后由谁代表贵司确认"系统达到要求、可以正式用了"？这位负责人最看重哪几件事做到了才算通过？（例：业务员能独立完成一次报价、旧客户和产品数据查得到、导出的报价单格式符合贵司习惯）
> 💬 验收是双方握手那一刻。提前约好"看哪几条算过"对您也是保护——我们照着交付，您照着收，不会出现做完了却迟迟定不下来。

**18.【标｜多选】** 报价单最终交到客户手上时，您希望它长什么样？
- 带公司抬头/Logo / 需要盖章或电子签 / 导出成 PDF / 导出 Excel / 中英文双语（外贸客户） / 按客户分币种（人民币/美元等） / 其他（请补充）
> 💬 报价单是直接给客户看的"门面"。问清格式、语言、币种，是想让系统出的单子拿出去就能用，不用业务员再手工二次加工。

**19.【标｜多选】** 上线后需要我们培训哪些岗位的同事？更习惯哪种方式？另外贵司谁会当系统"管理员"（加减员工账号、调权限）？
- 业务员（现场或线上讲一次） / 内勤·价格维护人员 / 管理者·老板 / 给一份图文操作手册自学即可 / 备注写下拟任管理员
> 💬 用得起来系统才有价值。提前知道培训给谁、谁当管理员，我们好准备合适材料、把管理权限交到对的人手上。

**20.【标｜单选】** 上线后，日常小事（改产品价格、加新客户、调报价单模板）您更希望贵司同事自己操作，还是交给我们？
- 大部分自己来、复杂的找你们 / 简单的自己来、价格·模板类找你们 / 基本都交给你们 / 还没想好，想听建议
> 💬 把"哪些您自己来、哪些找我们"提前说清，您心里有数，我们也好把上线后的陪跑安排到位。这也关系到后续运维的投入和成本预期。

**21.【标｜多选】** 您和业务员平时主要用什么设备打开系统？业务员需要在外面（客户现场、出差）用手机/平板临时查价报价吗？
- 办公室电脑(Windows) / 苹果电脑(Mac) / 手机·平板，在外跑客户时用 / 客户现场需当场出单 / 主要在公司内部用
> 💬 如果业务员常在外面用手机，我们在界面适配上要早做准备，免得做成只适合电脑、业务员在客户面前打不开。

## 六、长期托管与成本预期

**22.【必｜单选】** 为了给您配一套够用又不浪费的方案，先对一下预期：这类系统除了我们的服务费，每月还有一笔像房租一样的固定开销——云服务器和托管运维（行业里通常几百元每月起，按用量走）。您对这部分月度投入，大致哪个区间比较舒服？
- 几百元以内 / 五百到一千元 / 一千到两千元 / 按需就好、先看方案再定 / 希望您给个建议区间
> 💬 这是我们最想提前和您对齐的一题——把服务器+托管这类必然且持续的成本先摆明面，避免上线后第一张账单成"惊吓"。用区间而非问数字，也是想让您选起来更轻松。

**23.【标｜单选】** 万一系统临时用不了，大概多久会真正影响到您的生意？（并在备注写：一周里哪个时段报价/查货最忙、最不能断）
- 几分钟内必须恢复 / 半小时内可接受 / 半天内可接受 / 听你们建议
> 💬 万一真出故障，我们想知道您心里的底线，好决定配多强（也多贵）的容灾——别花冤枉钱，也别留隐患。报价数据我们已按"分钟级恢复、最多丢 15 分钟"设计，想和您的实际预期对齐。

**24.【标｜单选】** 万一系统出状况，您希望我们的技术支持在什么时段"随叫随到"？
- 工作日上班时间就够 / 工作日延长到晚上 / 含周末 / 全天候 7×24 / 按建议来
> 💬 全天候待命会增加成本，工作日服务更经济。我们想按您生意真实的繁忙节奏来定，既保关键时段、又不让您为用不上的时段多花钱。

**25.【标｜填空】** 万一系统出状况，您希望我们第一时间联系谁？请留 1-2 位对接人（姓名、岗位、电话/微信），并标出谁是出问题时能拍板的最终负责人。
> 💬 出事时最怕找不到人、耽误处理。先把联系人定好，真有急事我们能直接找对人，而不是层层转。

**26.【全｜单选】** 我们除了在云上每日备份，还建议在贵公司一台常开的办公电脑上自动留一份加密副本，相当于把最重要的数据多攥一份在自己手里（几乎不增加费用）。这个您方便提供电脑吗？
- 可以，能提供一台常开电脑 / 可以，但需要你们指导怎么弄 / 暂时不需要、云上备份就够 / 想听你们建议
> 💬 报价和客户数据是您的命根子。多一份握在自己手里的副本，是"就算云那边出极端意外，数据也丢不了"的最后一道保险。

**27.【全｜单选】** 上线后，您倾向我们持续帮您托管运维、有问题随时兜底，还是先用一段时间看看效果再说？没有标准答案，怎么舒服怎么来。
- 希望你们长期托管、随时兜底 / 先用一段看看效果再定 / 按效果分阶段决定 / 还没想，到时再说
> 💬 我们更希望陪您长期跑下去，而不是交付完就撒手。这题没标准答案，您怎么舒服怎么来。
