# 新手手把手：阿里云服务器部署 ProductHub（零基础·高颗粒）

> 适用：你已购 ECS（2核2G / Alibaba Cloud Linux / 华北2北京 / 已预装 Docker / 3M 固定带宽）。
> 全程不用记命令——照抄即可。每步都有「✅ 怎么算成功」和「⚠️ 坑」。卡住就截图发我。
> 顺序原则：**备案是长杆，先启动备案，等审核期间并行做服务器部署（用 IP 测），备案过了再绑域名+HTTPS。**

---

## 第 0 步 · 现在就启动两件长事（并行）
### 0.1 买域名
阿里云控制台 → 顶部搜「域名」→ **域名注册** → 想个名字（如 `hesheng-xxx.com`）→ 查可注 → 下单实名认证（企业用营业执照）。
> ✅ 成功：域名出现在「域名控制台」列表、状态正常。

### 0.2 启动 ICP 备案（决定多久能用域名对外）
阿里云控制台 → 搜「备案」→ **ICP 备案** → 开始备案 → 按引导填（主体=北京合胜营业执照、法人、网站负责人、用阿里云 App 扫脸核验）。
- 需要：营业执照、法人/网站负责人身份证、能接听的手机、一个「阿里云备案服务码」（购买 ECS 满足时长后在备案页申领）。
> ✅ 成功：备案进入「审核中」。⚠️ 备案没下来之前，**域名 + 80/443 端口对外是被运营商拦的**——所以下面先用 IP 测。

---

## 第 1 步 · 第一次连上你的服务器（最省事：网页版 Workbench，不用装任何工具）
### 1.1 找到服务器和它的公网 IP
控制台 → 搜「ECS」或「云服务器」→ 左侧「实例」→ 看到你买的那台 → 记下它的 **公网 IP**（一串数字，如 `47.x.x.x`）。

### 1.2 设一个登录密码
选中实例 → 右侧「…」更多 → 密码/密钥 → **重置实例密码** → 设一个你记得住的强密码（管理员用户名是 `root`）→ 按提示**重启实例**生效。
> ⚠️ 重置密码通常要重启才生效，按提示来。

### 1.3 用 Workbench 网页连进去
实例那一行 → **远程连接** → 选 **Workbench**（网页终端）→ 用户名 `root` + 刚设的密码 → 连接。
> ✅ 成功：浏览器里出现一个黑底命令行，提示符像 `[root@xxx ~]#`。这就是你在操作服务器了。
> 以后所有「敲命令」都在这个黑框里，一行一行复制粘贴 + 回车。

---

## 第 2 步 · 服务器基础体检与加固（连上后照抄）
### 2.1 确认 Docker 正常（预装的）
逐行粘贴回车：
```
docker --version
docker compose version
systemctl enable --now docker
docker ps
```
> ✅ 成功：前两行各打印一个版本号；`docker ps` 显示一个空表头（CONTAINER ID...）。
> ⚠️ 若 `docker compose version` 报错（没装 compose 插件），粘贴：`dnf install -y docker-compose-plugin` 再试。

### 2.2 加 4G Swap（2核2G 必做，否则后面会"卡死/构建失败"）
逐行粘贴：
```
dd if=/dev/zero of=/swapfile bs=1M count=4096
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
sysctl -w vm.swappiness=10
echo 'vm.swappiness=10' >> /etc/sysctl.conf
free -h
```
> ✅ 成功：`free -h` 最后一行 `Swap:` 显示约 `4.0Gi`。

### 2.3 收紧安全组（安全必做）==（TBD）==
控制台 → ECS → 实例 → 点进去 → 「安全组」→ 配置规则 → **入方向**：
- 保留/添加：**80**（HTTP，源 0.0.0.0/0）、**443**（HTTPS，源 0.0.0.0/0）
- **22**（SSH）：把「源」改成**只你自己的公网 IP**（浏览器搜"我的IP"得到，填 `你的IP/32`）
- 测试期临时加：**8080**（源**只填你自己的 IP**，备案前用它从浏览器测；测完删）
- 其余多余端口一律删除。**绝不开放 5432（数据库）。**
> ✅ 成功：入方向规则就剩上面这几条。

---

## 第 3 步 · 把项目代码放到服务器
代码要先上传到服务器才能部署。二选一：
- **方案 A（推荐，方便日后更新）**：先把项目推到一个**私有 Git 仓**（阿里云 Codeup 或 Gitee 私有库），再在服务器：
  ```
  mkdir -p /opt/producthub && cd /opt/producthub
  git clone <你的私有仓地址> .
  ```
- **方案 B（不想用 Git）**：把项目打成 zip，用 Workbench 的「上传文件」传到服务器，再 `unzip` 到 `/opt/producthub`。
> ⚠️ 目前代码还没提交到任何远程仓。要走方案 A，跟我说一声，我帮你把代码提交并推到你建好的私有仓（你给我仓地址）。
> ⚠️ 前端构建较吃内存，2核2G 上**靠第 2.2 步的 4G swap** 才能跑完（会慢些）；若失败我教你在本机预构建后上传镜像。

## 第 4 步 · 配置并启动（用单机自管方案）
在 `/opt/producthub` 下：
```
cp deploy/.env.prod.example deploy/.env.prod && chmod 600 deploy/.env.prod
cp deploy/pgbackrest.conf.example deploy/pgbackrest.conf
```
用 Workbench 编辑器（或 `vi`）打开 `deploy/.env.prod` 填：`POSTGRES_PASSWORD`（强随机）、`JWT_SECRET`（粘贴 `openssl rand -hex 24` 的输出）、`SITE_DOMAIN`（先填你的域名，备案前也可先随便填、不影响 IP 测试）。
启动：
```
docker compose --env-file deploy/.env.prod -f deploy/docker-compose.single.yml up -d --build
```
> ⚠️ 首次会构建镜像、跑数据库迁移，**耐心等几分钟**（前端构建最慢）。
初始化备份（一次性）：
```
docker compose -f deploy/docker-compose.single.yml exec postgres pgbackrest --stanza=producthub stanza-create
docker compose -f deploy/docker-compose.single.yml exec postgres pgbackrest --stanza=producthub check
docker compose -f deploy/docker-compose.single.yml exec postgres pgbackrest --stanza=producthub --type=full backup
```
> ✅ 成功：`docker compose ... ps` 看到 postgres/app/caddy 在跑、frontend 已 Exited(0)（正常，它是一次性构建容器）。

## 第 5 步 · 备案前怎么测（用 IP）
备案没下来不能用域名+80/443。临时办法（测完即收）：把 Caddy 暂时映射到 8080，或直接验证后端健康：
```
curl -fsS http://127.0.0.1:8000/healthz     # 在服务器里执行，返回 {"status":"ok"} 即后端正常
```
要在你自己电脑浏览器看页面：在安全组临时开 8080（限你的 IP），并把 Caddy 端口临时改 8080（我可给你这一步的小改动），浏览器访问 `http://公网IP:8080`。
> ✅ 成功：healthz 返回 ok；浏览器能打开登录页。

## 第 6 步 · 备案通过后 · 绑域名 + 自动 HTTPS
1. 控制台 → 域名 → 解析 → 添加记录：**A 记录**，主机记录 `@`（和 `www`），记录值 = 你的**公网 IP**。
2. 确认安全组 80/443 已开、域名解析已生效（`ping 你的域名` 能解析到 IP）。
3. `deploy/.env.prod` 的 `SITE_DOMAIN` 填正式域名 → 重启栈：`docker compose --env-file deploy/.env.prod -f deploy/docker-compose.single.yml up -d`。
4. Caddy 会**自动申请并续期 HTTPS 证书**。
> ✅ 成功：浏览器开 `https://你的域名` 出现登录页、地址栏是小锁（https）。

## 第 7 步 · 让备份/监控自动跑（上线后）
```
crontab -e
```
加（按 i 进入编辑、粘贴、Esc 后输 :wq 保存）：
```
0 1 * * 0  cd /opt/producthub && docker compose -f deploy/docker-compose.single.yml exec -T postgres pgbackrest --stanza=producthub --type=full backup
0 1 * * 1-6 cd /opt/producthub && docker compose -f deploy/docker-compose.single.yml exec -T postgres pgbackrest --stanza=producthub --type=incr backup
```
> 强烈建议在 `pgbackrest.conf` 启用 OSS 异地仓（备份推到对象存储，机器坏了备份还在）——这步我单独教你配。

## 卡住了怎么办（求助点）
任何一步报红/不符预期，**把命令和报错截图发我**。常见：
- Workbench 连不上 → 检查密码是否重置后重启了、实例是否「运行中」。
- `up -d --build` 卡住/被杀 → 多半是内存，确认第 2.2 步 swap 已开。
- 域名打不开但 IP 能开 → 多半是备案没过或解析没生效。
