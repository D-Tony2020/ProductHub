## 四、Moore 侧目标结构（映射目的地）
ProductHub 数据模型：**品类(整机/部件) → 属性=选项 → 配置树(白盒自配/黑盒外购件) → SKU(指纹唯一) → 供应商 → 报价(逐币种现价+历史)**。
- 结构化产品 → 品类模板 + 属性选项；每个具体规格组合 = 一个 SKU。
- 外购整件/部件 → 黑盒成品件（`供应商名|件名`）。
- 价格 → 逐币种现价（同日纠错 supersede，历史留痕）。
- 导入走 [存量数据导入模板.xlsx](templates/存量数据导入模板.xlsx)：固定列 + `attr:路径` + `part:槽路径`。

## 五、迁移与对账方案（拿到数据后执行，先在 staging 演练）
1. **映射**：旧字段 → 导入模板列（attr:/part:/价格），形成字段映射表。
2. **清洗**：单位/币种归一、枚举值对齐字典、去重、补缺、非法值剔除并登记。
3. **试导（dry-run）**：用导入端点 dry-run，零副作用看报告（每行 ok/error），修数据再导。
4. **正式导入（confirm）**：单事务全量重校验入库（TOCTOU 安全）。
5. **对账（验收硬指标）**：
   - 行数核对：旧系统各实体条数 == 导入成功条数 + 合理剔除数（剔除须可解释）。
   - 抽样核对：随机抽 N 条，逐字段比对旧↔新一致。
   - 金额合计核对：报价总额/分品类合计 旧↔新一致（含税口径统一后）。
   - 唯一性核对：指纹去重后无重复 SKU、无串档。
   - 出具《迁移对账报告》，双方签字方算迁移验收通过。
