# ProductHub 产品中台 — 部署与运维指南（单机 Docker/Compose）

> 适用：单节点容器部署（Caddy 自动 HTTPS + app + Postgres16 + 前端构建）。
> 目标主机建议：2C4G 以上 Linux（海外业务可选香港轻量云，免 ICP 备案）。
> 本指南覆盖：首次部署、升级发布、回滚与灾难恢复、备份/恢复演练、运维注意事项、排障。

---

## 0. 拓扑与文件
- 编排：[deploy/docker-compose.prod.yml](../deploy/docker-compose.prod.yml)（postgres + app + frontend 一次性构建 + caddy）
- 反代/HTTPS：[deploy/Caddyfile](../deploy/Caddyfile)（API `/api/*`+`/healthz` 反代 app:8000，其余走前端 SPA）
- 密钥模板：[deploy/.env.prod.example](../deploy/.env.prod.example)（复制为 `.env.prod` 填写，**永不入仓**）
- 备份：[deploy/backup.sh](../deploy/backup.sh)；恢复演练：[deploy/restore.sh](../deploy/restore.sh)
- 前端构建：[frontend/Dockerfile](../frontend/Dockerfile)（npm 编译 → 产物入共享卷 `frontend_dist`，Caddy 托管）

## 1. 前置检查
- Docker Engine ≥ 24 + Docker Compose v2（`docker compose version`）。
- 磁盘 ≥ 20G 空余（PG 数据 + 备份 + 镜像）。
- 域名已解析到主机公网 IP，**80/443 端口对外开放**（Caddy 走 ACME 自动签证书）。
- 确认 PG `max_connections`（默认 100）足够：当前 app 单实例 2 worker × 30 连接 = 60，余 40。**勿在单机起多个 app 实例**（见 §6）。

## 2. 首次部署
```sh
# 1) 取代码到 /opt/producthub（git clone 或上传）
cd /opt/producthub

# 2) 生成生产密钥文件（强随机；绝不复制 backend/.env 的 dev 值）
cp deploy/.env.prod.example deploy/.env.prod
#   填写：POSTGRES_PASSWORD（强随机）、JWT_SECRET（≥32 字节强随机，如 openssl rand -hex 24）、SITE_DOMAIN
chmod 600 deploy/.env.prod          # 仅属主可读，防共享主机泄露
chown "$(id -u):$(id -g)" deploy/.env.prod

# 3) 起栈（首启会构建后端/前端镜像、跑 alembic 迁移、签发证书）
docker compose --env-file deploy/.env.prod -f deploy/docker-compose.prod.yml up -d --build

# 4) 验证
curl -fsS https://<SITE_DOMAIN>/healthz        # 期望 {"status":"ok"}
docker compose -f deploy/docker-compose.prod.yml ps   # app/postgres/caddy 在跑；frontend 已 Exited(0)
docker compose -f deploy/docker-compose.prod.yml logs --tail=50 app   # 迁移 upgrade head 成功、无报错
```
> 说明：`frontend` 是一次性构建容器，编译完把产物写入共享卷后退出（状态 Exited 0 正常）；Caddy 用 `service_completed_successfully` 等它完成再起。

## 3. 升级发布（已上线后更新版本）
```sh
cd /opt/producthub && git pull            # 或上传新代码
# 发布前务必先备份（见 §5）
docker compose --env-file deploy/.env.prod -f deploy/docker-compose.prod.yml up -d --build
#   app 启动命令内含 `alembic upgrade head`，迁移自动执行；frontend 重新构建并刷新 dist 卷
curl -fsS https://<SITE_DOMAIN>/healthz
```
发布顺序由 compose 保证：postgres healthy → app（迁移+起服务）→ frontend 构建完成 → caddy。

## 4. 回滚与灾难恢复
**发布出问题（代码/迁移）回滚：**
```sh
# 1) 回到上一个可用提交
git checkout <上一个稳定 tag/commit>
# 2) 若新版本跑过迁移、需回退 schema（在 app 容器内）：
docker compose -f deploy/docker-compose.prod.yml exec app alembic downgrade -1   # 或 downgrade <目标 revision>
#    注：本项目迁移已抽查 downgrade 可逆（见验收报告 A4-MIGRATION-009），但回退前务必已备份。
# 3) 重建并起旧版本
docker compose --env-file deploy/.env.prod -f deploy/docker-compose.prod.yml up -d --build
curl -fsS https://<SITE_DOMAIN>/healthz
```
**数据损坏 — 从备份恢复进生产库（停服操作，谨慎）：**
```sh
docker compose -f deploy/docker-compose.prod.yml stop app caddy        # 先停写入方
docker compose -f deploy/docker-compose.prod.yml exec -T postgres \
  pg_restore -U producthub -d producthub --clean --if-exists --no-owner \
  < /opt/producthub/backups/producthub-<STAMP>.dump
docker compose -f deploy/docker-compose.prod.yml start app caddy
```
> 红线提醒：恢复进生产库会覆盖现有数据，仅在确认数据已损坏且备份可用（先跑 §5 演练）时执行。

## 5. 备份与恢复演练
- **每日备份**：`deploy/backup.sh`，crontab 装：`30 2 * * * /opt/producthub/deploy/backup.sh >> /opt/producthub/backups/backup.log 2>&1`，保留 30 天，建议异地再留存一份。
- **每月恢复演练**（强制，否则备份等于没有）：
```sh
/opt/producthub/deploy/restore.sh /opt/producthub/backups/producthub-<STAMP>.dump
#   恢复到隔离库 producthub_restore_test 并打印 sku 行数；与生产规模对照即验证备份可用，不触碰生产库。
```

## 6. 运维注意事项
- **连接池/并发**：`db.py` 每 worker = pool_size 10 + overflow 20 = 30 连接；compose 用 2 worker = 60。**单机勿增 app 实例或 worker 数使 总连接 > PG max_connections×0.8（80）**；需横向扩容时前置 PgBouncer（transaction 模式）或调高 `max_connections`。
- **日志**：应用输出结构化 JSON 到 stdout（`LOG_LEVEL` 环境变量控级，默认 INFO，生产可设 WARNING）；由 `docker compose logs` / 日志驱动收集转发 ELK/Datadog。
- **时区**：DB 与应用固定 `Asia/Shanghai`（现价/报价日期口径一致），**勿改容器 TZ**。
- **密钥**：`.env.prod` 仅 `chmod 600`、不入 git；轮换 JWT_SECRET 会使所有在用令牌失效（用户需重登）。
- **CORS**：生产由 Caddy 同源反代，前端与 API 同域；**务必经域名访问，勿直接暴露/访问 app:8000**。

## 7. 排障速查
| 症状 | 排查 |
|---|---|
| `/healthz` 不通 | `docker compose ps`；看 app 日志迁移是否失败；postgres 是否 healthy |
| 前端 404/空白 | `frontend` 容器是否 Exited(0)；`frontend_dist` 卷是否有 dist；Caddy 日志 |
| 502/连接耗尽 | PG 活跃连接数 `SELECT count(*) FROM pg_stat_activity`；是否多实例/worker 超限（§6） |
| 证书签发失败 | 80 端口是否对外开放；域名解析是否正确；Caddy 日志 ACME 报错 |
| 录价/报价 403 | 权限：用户 role/can_set_price（设计如此，见验收 Gate2 C3） |
