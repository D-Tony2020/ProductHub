# ProductHub 产品中台 · 产品说明指南

> 一页看懂每个界面"是什么、怎么用、要记住什么"。本指南收纳了原先散落在各页面上的说明性文字——界面本身保持干净，说明集中到这里。

---

## 一句话定位

把"灭火器有哪些型号、卖多少钱、能不能报价"管起来：**配出 SKU → 定价 → 进报价单**。每个 SKU 由其配置唯一确定，一经生成永不原地修改。

---

## 各界面速查

### 1. 产品库（首页）

业务员的主场，两个视角切换：

| 视角 | 是什么 | 怎么用 |
|---|---|---|
| **SKU 货架**（默认） | 业务员日常：找货 / 查价 / 加报价单 | 左侧按快捷视图、品类筛选；卡片/表格双视图；点卡片看详情、改价、加入报价单 |
| **产品全貌** | 公司有哪些产品（按品类聚合，比 SKU 粗一档） | 只读总览；点任一品类卡 → 下钻到该品类的 SKU 货架 |

- **已作废 SKU**：自动灰显并沉到列表末尾，不主动占用注意力，但仍可点开查看。
- 改产品类型的结构（属性、部件槽）请到 **系统设置 · 产品模板**。

### 2. 配置看板

把一台灭火器"配"出来，得到唯一 SKU。

- **起点**：选一个品类卡开始。整机品类可「逐项配置」或「整机直采」；单卖配件「开始配置」。
  - 卡片可在组内自由拖动排序（仅本机记忆，不影响他人）。
  - 已停用品类灰显沉底、不可新建。
- **逐项配置**：左=产品构成树，中=当前节点编辑，右=实时摘要与命中。
- **完整即查重**：配置一完整就自动比对既有 SKU——
  - 命中既有 → 只给「查看 / 加入报价单」，不重复建。
  - 全新 → 「保存为新 SKU」，获得唯一编码。
- **整机直采**：整台外购成品直接作为可售 SKU，无需逐项配置；其规格随采购件灰盒带入，不进指纹、不参与报价。
- **修改既有 SKU**：保存会生成一个**新** SKU（指纹改变，原 SKU 绝不原地修改），原 SKU 由你选择停用或保活。

### 3. 报价单

把在售、已定价的 SKU 组成对外报价。价格变动行会高亮提示，结算栏常驻底部。

### 4. 供应商与采购件

管理供应商及其采购件（整机/部件）；可补录采购件的内部规格（全选填、仅描述，不进 SKU 指纹）。

### 5. 系统设置 · 产品模板（管理员）

给"产品长什么样"建模：**节点类型 / 规格属性与选项 / 部件槽**。

- 左栏按 整机品类 / 可单卖配件 / 通用部件 分组；支持搜索定位与组内拖拽排序（搜索时暂停）。
- 徽标：绿=在售 SKU 数，橙↑=被几个上级引用，"未挂载"=未被任何上级引用。
- 节点过长时列表区独立滚动，搜索与说明常驻。

---

## 三条不可逾越的纪律（数据红线）

1. **编码/指纹不可变**：SKU 编码与配置指纹一经生成永不原地修改。改配置 = 生成新 SKU。
2. **code 后端自动生成且不可变**：节点类型、属性、选项的 code 由后端按名称自动生成查重，界面只展示名称，不展示 code。
3. **软停用，不物理删除**：停用只影响"新配置不可再选"，**既有 SKU 完全不受影响**。选项改名仅用于纠错（错别字/规范表述）；若是换义（如 4kg→5kg），应新建选项并停用旧选项。
